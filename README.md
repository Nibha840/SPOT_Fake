# Spot the Fake Photo — How to run this

## Step 1 — Take your photos (~30-60 min, as the brief says)

Using your phone, take:
- **~50 "real" photos** — normal photos of real things around you (objects,
  people, rooms, outdoors, your hand, anything). Vary lighting and angle.
- **~50 "screen" photos** — point your phone camera at ANOTHER screen (a
  laptop, another phone, a TV) showing a picture, and photograph that
  screen. Also try a few photos of a printed-out picture/printout if you can.
  Vary: which screen, brightness, angle, distance, indoor/outdoor lighting.

Put them here:
```
spot_fake/
  real/      <- your ~50 real photos go here (.jpg/.png, any names)
  screen/    <- your ~50 screen photos go here
```

## Step 2 — Train

```bash
cd spot_fake
pip install pillow numpy scipy scikit-learn   # only needed for training
python train.py
```

This prints your **held-out test accuracy** (the honest number to report),
a confusion matrix, and which files were misclassified — open those and see
if you can tell why (often: glare-free screen photo, or a very low-light
real photo). It saves `model.json`.

## Step 3 — Predict on any image

```bash
python predict.py some_image.jpg
python predict.py "real\WhatsApp Image 2026-06-29 at 19.27.32 (13).jpeg"
```

Prints one number 0–1. (1 = looks like a screen recapture.)

`predict.py` only needs `pillow`, `numpy`, and `scipy` — **not**
scikit-learn — since the trained weights are baked into `model.json` as
plain numbers. This keeps it light for eventual phone deployment.

## Files

- `features.py` — turns one image into ~32 numbers (the actual "detector logic": Nyquist-preserved FFT/moire energy, custom LBP texture histogram, color cast, sharpness uniformity, glare, border edges). Read the docstring and NOTE.md for details.
- `train.py` — reads `real/` + `screen/`, extracts features, compares Logistic Regression, SVM (RBF), and Random Forest, and saves the best model to `model.json`.
- `predict.py` — the required one-line predictor. Loads `model.json`, runs feature extraction, and scores one image. No heavy ML library needed at inference time.
- `model.json` — generated by `train.py`. **You must run train.py before
  predict.py will work** (it doesn't exist until you do, since it depends on
  your photos).
- `NOTE.md` — the half-page write-up to send along with your code. Fill in
  the accuracy/latency placeholders with your real numbers after running
  `train.py` and timing `predict.py` on your machine.

## After training: fill in NOTE.md

1. Copy the **held-out test accuracy** that `train.py` prints into `NOTE.md`.
2. Time `predict.py` on your own laptop:
   ```bash
   python -c "import time; from predict import predict; predict('real/whatever.jpg'); t=time.time(); [predict('real/whatever.jpg') for _ in range(10)]; print((time.time()-t)/10*1000, 'ms')"
   ```
   Put that ms number + "laptop CPU" (or whatever device) into `NOTE.md`.
3. If accuracy is under 95%: open the misclassified files train.py lists,
   take a few more screen photos similar to the ones it's getting wrong
   (more lighting/angle variety usually fixes this), and rerun `train.py`.

## Live camera demo (Optional & Impressive)

We have built a beautiful local web server to test this model live with your camera:

```bash
python app.py
```

This starts a server at **http://localhost:8080**. Open it in your browser, allow camera permissions, and you can test the recaptured-screen detector live on your webcam! It features:
- **Scan Frame**: Capture a single snapshot and score it instantly.
- **Auto-Scan**: Capture and analyze frames continuously every second to see the score change in real-time.
- **Aesthetic Glassmorphism UI**: Beautiful gauges and visual indicators showing if the feed is `REAL` (green) or a `SCREEN` (red).
